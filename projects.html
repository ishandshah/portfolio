Oracle RMS CDC to Kafka for Cost Accounting

Ishan built high-throughput CDC and Kafka pipelines to stream inventory and cost-related events from Oracle RMS systems into downstream financial and analytical platforms. The work included schema design, partitioning, ordering, idempotency, history replay, state stores, and event contracts supporting item-level cost, stock-on-hand visibility, invoice-level analysis, and stock ledger transactions.

Role
Streaming data engineer for source capture, topic design, replay behavior, and consumer contracts.
Artifacts
CDC pipeline design, Kafka schemas, partitioning strategy, idempotency handling, state stores, and replay support.
Evidence signal
Finance-facing data movement from legacy retail systems into event-driven analytical workflows.
  • Improved near-real-time cost and inventory visibility.
  • Supported finance and accounting reconciliation workflows.
  • Reduced reliance on delayed batch-style processing patterns and supported lower-lag downstream consumption.

Security analytics infrastructure supporting high-volume event search, including 500K events per second indexing and 95% uptime targets.

  • Supported large-scale event indexing and security analytics.
  • Strengthened reliability of high-throughput search infrastructure.
